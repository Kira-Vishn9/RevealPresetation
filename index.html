What is D3.js?

is a JavaScript library used to create interactive renderings in the browser

What opportunities does it give?

allows us to manipulate web page elements in the context of a dataset

What things can you work with?

These elements can be HTML, SVG, or Canvas elements and can be inserted, removed, or edited according to the contents of the dataset. It is a library for manipulating DOM objects

D3.js Features

  • Extremely flexible.
  • Easy to use and fast.
  • Supports large datasets.
  • Declarative programming.
  • Code reuse.
  • Has a wide range of curve generation functions.
  • Associates data with an element or group of elements in an html page.

Benefits of D3.js

D3.js is an open source project and works without any plugin. This requires very less code and provides the following benefits:
  • Excellent data visualization.
  • It's modular. You can download a small piece of D3.js that you want to use. There is no need to download the entire library each time.
  • It is easy to build a charting component.
  • DOM manipulation.

    • before creating the first graph, let's analyze D3.js - Elections

    D3.js helps you select elements from an HTML page using the following two methods:

    1. select ()
    2. selectAll ()

    select () ?

    selects only one DOM element by matching the given CSS selector. If there is more than one element for a given CSS selector, it only selects the first one.

    • works the same way as -document.querySelector('.xxx') in our usual js

    selectAll () ?

    selects all DOM elements by matching the given CSS selector. If you're familiar with selecting elements with jQuery, D3.js selectors are pretty much the same.

    • works the same way as -document.querySelectorAll('.xxx') in our usual js

    select() method

    The select() method selects an HTML element based on CSS selectors. In CSS selectors, you can define and access HTML elements in the following three ways:

    • HTML element tag (e.g. div, h1, p, span, etc.)
    • HTML element class name
    • HTML element ID

    visual example

  • d3.select(“div“)
  • d3.select(“.class name”)
  • d3.select(“#id”)

    • Adding DOM Elements

    • append ()
      • and
    • text ()

    append() method

    The append() method adds a new element as the last child of the element in the current selection

    Text() Method

    The text() method is used to set the content of the selected/added elements.

    works like document.querySelector('span').innerHTML = ('')

    attr() method

    attr() method is used to add or update attribute of selected elements

    Style() Method

    The style() method is used to set the style property of the selected elements

    classed ()

    • Add Class - To add a class, the second parameter of the classified method must be set to true
    • Delete Class - To delete a class, the second parameter of the classified method must be set to false

    visual example

    • d3.select(".myclass").classed("myanotherclass", true);
    • d3.select(".myclass").classed("myanotherclass", false);

    Class Check

    Class Check - To check for the existence of a class, simply omit the second parameter and pass in the name of the class you are accessing. This will return true if it exists, false if it doesn't.

    Example: d3.select(".myclass").classed("myanotherclass");

    D3.js — Data Join

    By default, D3.js gives the dataset the highest priority in its methods, and each element in the dataset corresponds to an HTML element.

    The main purpose of data

    aggregation is to match the elements of an existing document with a given set of data. It creates a virtual representation of a document relative to a given set of data and provides methods for working with the virtual representation

    The code to create a new li element looks like this:

    d3.select("#list").selectAll("li")
   .data([10, 20, 30, 25, 15])
   .text(function(d) { return "This is pre-existing element and the value is " + d; })
   .enter()
   .append("li")
   .text(function(d) 
      { return "This is dynamically created element and the value is " + d; });

    Data attachment provides another method called the exit() method to handle data items that are dynamically removed from the dataset as shown below.

    d3.selectAll("li")
   .data([10, 20, 30, 15])
   .exit()
   .remove()

    Here we have removed the fourth element from the dataset and its corresponding li using the exit() and remove() methods.

    The complete code looks like this -- 

    <!DOCTYPE html>
<html>
   <head>
      <script type = "text/javascript" src = "https://d3js.org/d3.v4.min.js"></script>
      <style>
         body { font-family: Arial; }
      </style>
   </head>

   <body>
      <ul id = "list">
         <li></li>
         <li></li>
      </ul>
        
      <input type = "button" name = "remove" value = "Remove fourth value" 
         onclick = "javascript:remove()" />
      
      <script>
         d3.select("#list").selectAll("li")
            .data([10, 20, 30, 25, 15])
            .text(function(d) 
               { return "This is pre-existing element and the value is " + d; })
            .enter()
            .append("li")
            .text(function(d) 
               { return "This is dynamically created element and the value is " + d; });
             
         function remove() {
            d3.selectAll("li")
            .data([10, 20, 30, 15])
            .exit()
            .remove()
         }
      </script>
   </body>
</html>

    Data aggregation methods

  • datum() method
  • data () method
  • enter () method
  • exit () method

    • Briefly about these methods

  • The data() method is used to assign a dataset to a collection of elements in an HTML document
  • The enter() method outputs a set of data elements for which there was no graphical element previously.
  • The exit() method displays a set of graphic elements for which there is no more data.
  • The datum() method is used to set the value for a single element in an HTML document

    Features of SVG

    • SVG is a vector image format based on text
    • SVG is similar in structure to HTML.
    • SVG can be represented as a Document object model.
    • SVG properties can be specified as attributes.
    • The SVG must have absolute positions relative to the origin (0, 0).
    • SVG can be included as is in an HTML document.
